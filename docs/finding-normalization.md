# Finding Normalization

Security scanners produce different output formats. The analyzer needs a common schema before it can compare, correlate, or score findings from multiple tools.

The current normalization steps support Semgrep, Checkov, Gitleaks, and Trivy JSON output.

## Input

Scanners write raw output to:

```text
scanner-results/semgrep.json
scanner-results/checkov.json
scanner-results/gitleaks.json
scanner-results/trivy.json
```

These files are generated by the CI workflow or by local scanner runs.

## Output

The analyzer writes normalized findings to:

```text
scanner-results/normalized-findings.json
```

The output file is generated runtime data and is not committed to Git.

## Common Finding Fields

The initial normalized schema keeps the fields needed for risk analysis:

| Field | Purpose |
| --- | --- |
| `scanner` | Source tool, such as `semgrep`. |
| `rule_id` | Scanner rule or check identifier. |
| `title` | Human-readable finding message. |
| `severity` | Normalized severity such as `low`, `medium`, or `high`. |
| `category` | Security domain, such as `sast`, `iac`, `secret`, or `sca`. |
| `file` | File where the finding was detected. |
| `line` | Starting line of the finding. |
| `end_line` | Ending line, if available. |
| `cwe` | Related CWE identifiers. |
| `owasp` | Related OWASP category mappings. |
| `confidence` | Scanner confidence signal. |
| `impact` | Estimated impact signal. |
| `likelihood` | Estimated likelihood signal. |

## Run

Normalize Semgrep:

```powershell
.\.venv\Scripts\python.exe -m analyzer.normalize_semgrep --output scanner-results/semgrep-normalized-findings.json
```

Normalize Checkov:

```powershell
.\.venv\Scripts\python.exe -m analyzer.normalize_checkov --output scanner-results/checkov-normalized-findings.json
```

Normalize Gitleaks:

```powershell
.\.venv\Scripts\python.exe -m analyzer.normalize_gitleaks --output scanner-results/gitleaks-normalized-findings.json
```

Normalize Trivy:

```powershell
.\.venv\Scripts\python.exe -m analyzer.normalize_trivy --output scanner-results/trivy-normalized-findings.json
```

Merge normalized scanner outputs:

```powershell
.\.venv\Scripts\python.exe -m analyzer.merge_findings --input scanner-results/semgrep-normalized-findings.json scanner-results/checkov-normalized-findings.json scanner-results/gitleaks-normalized-findings.json scanner-results/trivy-normalized-findings.json
```

Expected output:

```text
Merged findings are written to scanner-results/normalized-findings.json
```

## Why This Matters

Raw scanner output is useful, but each tool speaks its own format. Normalization creates a stable internal representation that scoring, correlation, and reporting can use regardless of whether the finding came from Semgrep, Gitleaks, Trivy, Checkov, or another scanner.
